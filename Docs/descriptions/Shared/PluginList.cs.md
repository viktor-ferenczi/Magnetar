# Shared/PluginList.cs

**Project:** Shared · **Namespace:** `Pulsar.Shared` · **Kind:** class · **Lines:** 842

## Summary
The plugin catalog. Aggregates plugin metadata (`PluginData`) from five sources — remote GitHub "hubs" (repos containing many plugin XML descriptors), single remote GitHub plugins, Steam Workshop mods, local hub folders, and loose local DLLs/dev folders — into one queryable, enumerable collection keyed by plugin id. It manages on-disk `.bin` caches of downloaded whitelists (ProtoBuf-serialized), invalidates them by repo commit hash and a `MaxSourceAge` TTL, and resolves plugin groups and transitive dependencies. Hubs are read from XML (`XmlSerializer`); DLLs are sniffed via Mono.Cecil to skip native assemblies.

## Types
### `PluginList` — class, public : `IEnumerable<PluginData>`
Holds separate dictionaries per source category and exposes a merged, de-duplicated view via the private `Plugins` property (precedence: pure remote plugins, then mods, local, local hubs, remote hubs; first wins on id collision). Construction discovers and caches everything; the `Update*` methods refresh sources incrementally.
- **Fields:** `remoteHubPlugins` / `localHubPlugins` — per-source-key maps of id->`PluginData`; `modPlugins`, `purePlugins`, `localPlugins` — flat id->`PluginData` maps; `SourcesConfig`, `ProfilesConfig` — config references; `SourceDir`, `LocalPluginDir` — root dirs.
- **Properties:** `Plugins` (private) — merged de-duplicated dictionary; `Count` — number of distinct plugins; `this[string key]` — `PluginData` by id; `this[Profile key]` — plugins whose id is in the profile; `PluginSourceDir`/`HubSourceDir` — cache subfolders under `Sources`.
- **Methods:** `Contains(string id)` / `TryGetPlugin(string id, out PluginData)` — id lookups; `PluginList(string mainDirectory, SourcesConfig, ProfilesConfig)` — ensures cache dirs, initializes remote list, finds local plugins, saves sources, resolves groups and dependencies; `GetModPlugins(Profile, HashSet<ulong> ignore)` — enabled `ModPlugin`s not in the ignore set that exist and target a supported runtime; `LoadPluginData` — applies per-plugin profile config; `FindPluginGroups` — links plugins sharing a `GroupId` into each other's `Group` lists; `FindPluginDependencies` — DFS over `DependencyIds` to populate each plugin's transitive `Dependencies`; `InitRemoteHub`/`UpdateRemoteHub` — fetch/cache a hub repo (hash + TTL gated, download-vs-read-cache fallbacks); `InitRemotePlugin`/`UpdateRemotePlugin` — same for a single-plugin repo; `AddHubPluginData` — load+label a hub's plugins into a dict; `AddMod`/`AddLocalPlugin`/`AddLocalHub`/`UpdateLocalHub` — register mods, dev folders, and local hub folders (folder hashed via `Tools.GetFolderHash`); `EnsureDirectories`; `InitRemoteList` — initial pass over enabled remote/mod sources; `UpdateRemoteList(bool force)` / `UpdateLocalList()` — incremental refresh, adding new and pruning removed sources; `IsNativeAssembly(string dll)` — true if Cecil throws `BadImageFormatException`; `TryReadHubFile`/`TryDownloadHubFile`/`TryReadPluginFile`/`TryDownloadPluginFile`/`TryLoadLocalHub`/`TrySaveFile` — ProtoBuf cache I/O and GitHub archive/file download/XML parse helpers (filtering `ObsoletePlugin` entries on read); `FindLocalPlugins` — scans `Local` dir DLLs plus local plugin/hub sources; `GetEnumerator` (both) — enumerates the merged plugin values.
